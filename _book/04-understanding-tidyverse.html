5  Understanding Tidyverse Functions

So far we used mutate() from dplyr, unnest() from tidyr, read_csv() from readr, map() from purrr, as_tibble() from tibble, and str_to_title() from stringr. Since we are using so many tidyverse packages, we can simply load them all via library(tidyverse).

mutate()

We already went over mutate(). This function is used to both create new columns and overwrite existing columns.

We can overwrite an existing column like so:

df_mutated <- mutate(df_unnested, year = as.numeric(year))

Having a numeric year is useful if, for example, we want to filter the data by years between 1960 and 1980. This is done with filter().

df_filtered <- filter(df_mutated, 1960 < year & year < 1980)

Pumping with |> or the older %>%

If you look at the examples in the mutate() documentation, you will see the symbol %>%.

%>% and |> are called pipes. They are very simple. They simply send the left side to the right side. The |> is newer and recommended because, where as the left side is an input, the right side is a function, and |> requires you to write () after the function name. Since |> enforces an explicit and safer style, let’s stick with it.

The other property to remember is that the pipe holds mathematical priority before brackets and exponents. So if you know BEDMASS, then you also know BE|>DMASS.

sqrt(5) 
[1] 2.236068

is the same as

5 |> sqrt()
[1] 2.236068

But

5 * 5 |> sqrt()
[1] 11.18034

is not the same as

(5 * 5) |> sqrt()
[1] 5

For a more relevant example,

df_mutated <- mutate(df_unnested, year = as.numeric(year))

is the same as

df_mutated <- df_unnested |> mutate(year = as.numeric(year))

which is the same as

df_mutated <- df_unnested |> mutate(year = year |> as.numeric())

Grouping with group

You may have also noticed group_by() in the filter() documentation.

group_by() groups the data for the function(s) that follow group_by(). It’s easiest to understand with an example:

df_mutated |> 
  group_by(country) |> 
  mutate(max_year = max(year)) |> 
  select(country, year, max_year)
# A tibble: 24 × 3
# Groups:   country [2]
   country      year max_year
   <chr>       <dbl>    <dbl>
 1 afghanistan  1952     2007
 2 afghanistan  1957     2007
 3 afghanistan  1962     2007
 4 afghanistan  1967     2007
 5 afghanistan  1972     2007
 6 afghanistan  1977     2007
 7 afghanistan  1982     2007
 8 afghanistan  1987     2007
 9 afghanistan  1992     2007
10 afghanistan  1997     2007
# ℹ 14 more rows

Notice how the functions like group_by() are on separate lines after the pumps. This is to improve readability as opposed to:

df_mutated |> group_by(country) |> mutate(max_year = max(year)) |> select(country, year, max_year)
